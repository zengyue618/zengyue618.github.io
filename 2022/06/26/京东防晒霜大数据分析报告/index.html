大数据架构下的爬虫数据分析

一：前言

此项目数据来源于京东防晒霜的商品评论页面，覆盖18个知名品牌的18000条评论数据，用自然语言处理技术，对评论内容进行情感分析，此外还采集了一些其它相关指标。

项目中广泛应用了各种数据分析以及大数据技术框架，包括pandas、mysql、Hadoop、Hive、Spark、DataX、Clickhouse，旨在将之前学习过的技术框架，融入其中，真正做到学以致用。

五：Spark on Hive构建数仓

SparkSql基于内存计算，相较于基于mapreduce的HiveSql，性能更强大。但Spark无源数据管理，因此，可以借用Hive的源数据管理功能。这就是Spark on Hive的模式——用Spark算，用Hive存。

用数仓理论的眼光看，刚才从mysql中直接导出到hive的数据，就算是ODS层

那么用SparkSql执行以下语句，创建这些中间表，就算是DWS层和APP层了

use sparkfinalhomework;

create table brand_vip as (
    select brand, count(*) as vip_count
    from sunblock
    where vip = "True"
    group by brand
);

create table total_vip as (
    select count(*) as count
    from sunblock
    group by vip
);

create table brand_sentiment as (
    select brand, avg(sentiment) as sentiment_avg
    from sunblock
    group by brand
);

create table brand_vip_level_sentiment as (
    select brand, avg(level) as vip_level, avg(sentiment) as vip_sentiment
    from sunblock
    where vip = "True"
    group by brand
);

create table goods_sentiment as (
    select goods, avg(sentiment) as sentiment_avg
    from sunblock
    group by goods
);

create table brand_level as (
    select brand, avg(level) as level
    from sunblock
    group by brand
);

create table goods_level as (
    select goods, avg(level) as level
    from sunblock
    group by goods
);

create table time_sentiment as (
    select `time`, sentiment
    from sunblock
    order by `time`
);

create table likes_comments as (
    select sum(like_value) as total_like, sum(comment_value) as total_comments, count(*) total
    from sunblock
);

create table day_comment as (
    select day, count(*) as count
    from sunblock
    group by day
    order by day
);

create table vip_level_sentiment as (
    select vip, avg(level) as level_avg, avg(sentiment) as sentiment_avg
    from sunblock
    group by vip
);

create table about_brand as (
    select bv.brand, bv.vip_count, bs.sentiment_avg, bl.level, bvls.vip_level, bvls.vip_sentiment
    from brand_vip bv
        join brand_sentiment bs on bv.brand = bs.brand
        join brand_level bl on bs.brand = bl.brand
        join brand_vip_level_sentiment bvls on bl.brand = bvls.brand
);

create table about_goods as (
    select gl.goods, gl.level, gs.sentiment_avg
    from goods_level gl join goods_sentiment gs on gl.goods = gs.goods
);

六：实现即席查询

刚才用Spark计算得到APP层这种可以直接用于查询分析的数据了，可是Spark依然满足不了即席查询的需求，所以我选择了更为强大的OLAP引擎——ClickHouse，它是俄罗斯开源的一款数据库软件，以性能强大闻名于世。

首先，依然是用dataX将hive表中的数据导入clickhouse中，当然，得要先在clickhouse中有相对应的表

以about_brand表为例，创建表

创建完了就可以用DataX导数据啦，这次DataX的配置文件是这样的

{
    "job": {
        "content": [
            {
                "reader": {
                    "name": "hdfsreader",
                    "parameter": {
                    "column": ["*"],
                        "defaultFS": "hdfs://node1:8020",
                        "encoding": "UTF-8",
                        "fieldDelimiter": "\001",
                        "fileType": "textfile",
                        "path": "/user/hive/warehouse/sparkfinalhomework.db/about_brand/*"
                    }
                },
                "writer": {
                    "name": "clickhousewriter",
                    "parameter": {
                        "batchByteSize": 134217728,
                        "batchSize": 65536,
                        "column": [
                            "brand",
                            "vip_count",
                            "sentiment_avg",
                            "level",
                            "vip_level",
                            "vip_sentiment"
                        ],
                        "connection": [
                            {
                                "jdbcUrl": "jdbc:clickhouse://node1:8123/JD_goods",
                                "table": [
                                    "about_brand",
                                ]
                            }
                        ],
                        "dryRun": false,
                        "password": "",
                        "postSql": [],
                        "preSql": [],
                        "username": "",

                    }
                }
            }
        ],
        "setting": {
            "speed": {
                "channel": ""
            }
        }
    }
}

执行dataX的命令，就可以导入成功
